# Contributing to MedNode

Thanks for helping build MedNode.

## Current Development Phase
This repository is currently in foundation mode. Focus contributions on:
- Architecture docs
- Interface contracts
- ADR proposals
- Repository standards and process

## How to Contribute
1. Open an issue describing the proposal.
2. For architecture-impacting changes, add or update an ADR in `docs/adr/`.
3. Keep pull requests focused and small.
4. Include rationale in the PR description.

## Branch and PR Guidelines
- Use short, descriptive branch names.
- Keep one concern per PR.
- Link related issues.
- Request review after documentation is complete and lint checks pass (once CI is enabled).

## Commit Style
Prefer Conventional Commits:
- `feat:`
- `fix:`
- `docs:`
- `chore:`
- `refactor:`

## Code of Conduct
By participating, you agree to follow `CODE_OF_CONDUCT.md`.
