# ADRs (Architecture Decision Records)

Store one file per architectural decision in this directory.

## Naming
Use:
`NNNN-short-title.md`

Example:
`0001-canonical-event-model.md`

## ADR Template
Each ADR should include:
1. Context
2. Decision
3. Consequences
4. Alternatives Considered

## Status Labels
- Proposed
- Accepted
- Superseded
- Rejected
